# 中国語 表現図鑑 — 運用マニュアル

このフォルダがサイト一式です。`index.html` は基本もう触りません。
ふだんの更新は **「videos.csv に1行足す」＋「slides/ に画像フォルダを置く」** だけです。

> 公開先は GitHub。Mac内のフォルダを更新しても公開サイトには反映されません。
> 編集した videos.csv と画像を **GitHubにアップロードして初めて反映** されます（下の「公開先への反映」参照）。

---

## フォルダの中身

```
中国語表現図鑑/
├─ index.html      ← サイト本体（触らない）
├─ videos.csv      ← これだけ編集する（動画一覧データ）
├─ slides/         ← スライド画像を「動画ごとのフォルダ」で入れる
│   └─ hen/            ← 1動画＝1フォルダ
│       ├─ 001.png
│       ├─ 002.png
│       └─ 003.png
└─ README.md       ← この説明書
```

> `slides/hen/` は動作確認用サンプルです。確認が済んだら消してOK
> （消したら videos.csv の1行目の「スライドフォルダ」「スライド枚数」も空にしてください）。

---

## スライドは「複数枚」めくれます

カードをタップすると、その動画のスライドを **スワイプ（スマホ）／矢印キー（PC）** でめくれます。
1動画につき、スライドを何枚でも登録できます。

## ふだんの更新作業（動画を出したとき）

### 手順1｜Keynoteから画像を書き出す

1. Keynote →「ファイル」→「書き出す」→「画像」タブ
2. スライド：**すべて** ／ フォーマット：**PNG**
3. 書き出すと `〇〇.001.png` `〇〇.002.png` … のように、
   **資料名＋3桁連番** で出てきます（Keynoteが自動で3桁番号を付けます）

### 手順2｜ファイル名の「資料名.」部分を一括で消す（数秒）

番号は既に付いているので、頭の文字だけ消して `001.png 002.png …` にします。

1. 書き出した画像を**全部選択**（⌘+A）
2. 右クリック →「**◯項目の名前を変更...**」
3. 小窓の左上を「**テキストを置換**」にする
4. 設定：
   - 検索文字列：頭の `資料名.`（**最後のドットまで含める**。例 `很挺的非常特别_解説.`）
   - 置換文字列：**空欄**
5. プレビューが `001.png 002.png …` になるのを確認
6. 「**名前を変更**」

> 検索文字列がうまく打てないときは、ファイル名をコピーして貼り付けると確実です。

### 手順3｜動画ごとのフォルダに入れて slides/ へ

1. `slides/` の中に、その動画用のフォルダを作る（例：`le`、`lihai` など半角英数）
2. リネームした画像をそのフォルダに入れる

```
slides/le/001.png, 002.png, 003.png …
```

### 手順4｜videos.csv に1行足す

`videos.csv` を開いて、一番下に1行追加します。列は以下の通り：

| 列名 | 入れるもの | 例 |
|---|---|---|
| タイトル | 動画タイトル | 【中国語】〜の使い分け |
| キーワード | 中国語を ` / ` で区切る | `了 / 过` |
| 要点 | ひとことメモ | 完了の「了」と経験の「过」… |
| ジャンル | 使い分け / 文法 / 語彙 / 旅行 | 文法 |
| 動画URL | YouTubeのURL | https://www.youtube.com/watch?v=... |
| 公開日 | YYYY-MM-DD | 2026-06-21 |
| **スライドフォルダ** | slides/ の中のフォルダ名 | le |
| **スライド枚数** | そのフォルダの画像枚数 | 3 |

- **キーワードが空の行は、サイトに表示されません**（vlog・雑談回はここを空に）
- **スライドフォルダ・枚数が空の行は、プレースホルダー（文字タイル）で表示**されます。あとから画像を用意して書き足せば画像に変わります

### 手順5｜公開先へ反映（GitHubにアップ）

下の「公開先への反映」を参照。1〜2分でサイトに反映されます。

---

## 公開先への反映（GitHub）

ふだんはこの2つをGitHubに上げれば反映されます。

**A. videos.csv を更新（上書き）**
- リポジトリで `videos.csv` を開く → 鉛筆マークで直接編集 → Commit
- または「Add file」→「Upload files」で、Mac側の新しい videos.csv をドラッグ（同名なので上書き）

**B. 画像フォルダを追加**
- リポジトリで `slides` を開く → 「Add file」→「Upload files」
- Mac側の動画フォルダ（例 `le`）ごとドラッグ＆ドロップ → Commit

> まとめて更新するときは、Mac側で videos.csv と画像を仕上げてから、
> 「Upload files」で **CSV（上書き）＋ 画像フォルダ（追加）を一度に** ドラッグするのが一番ラクです。

## リクエスト機能（解説してほしい表現の募集）

「リクエストを送る」ボタンが2か所あります（下部の常設バナー／検索0件時）。

### Googleフォームの設定方法

1. Googleフォームを新規作成（無料）。質問は自由記述ひとつでOK
2. 「送信」→ リンクアイコン → URLをコピー
3. `index.html` の上の方のこの行を書き換える：

```
const REQUEST_FORM_URL = "#";
```
↓
```
const REQUEST_FORM_URL = "https://forms.gle/xxxxxxxx";
```

4. 保存してアップ。

### 通知（誰かが送ったら気づくには）

フォーム編集 →「回答」タブ → 右上 ⋮ →「新しい回答についてのメール通知を受け取る」にチェック。

## Macで今すぐ見た目を確認したいとき

`index.html` をダブルクリックで開くと videos.csv を読み込めません（ブラウザ制限・正常）。
このフォルダで簡易サーバーを立てて確認します：

1. ターミナルを開く
2. `cd ` のあとにこのフォルダをドラッグ → Enter
3. `python3 -m http.server 8000`
4. ブラウザで `http://localhost:8000`
5. 終わったら Ctrl+C

※ iPhoneでの確認は、GitHubにアップして公開URLを開く形になります。

## 困ったとき

- **カードが表示されない** → サーバー経由で開いているか／videos.csv が同じ場所にあるか
- **画像が出ない** →
  - フォルダ名がCSVの「スライドフォルダ」と一致しているか（大文字小文字も区別）
  - 画像名が `001.png 002.png …`（3桁・連番）になっているか
  - 「スライド枚数」が実際の枚数と合っているか
- **特定の動画が出ない** → キーワード列が空欄になっていないか
